# Introduction Opening V2

Execution-based benchmarks have made few-shot code generation increasingly measurable: generated programs can be evaluated by running tests rather than relying only on surface-form similarity. At the same time, structure-aware systems often add syntax-aware retrieval, program-structure summaries, or structural hints that may help a model propose correct solutions. The difficulty is attribution. A pipeline can add structural prompting while also changing the number of generated candidates, the retrieval information available to episode construction, the quality of the induced prior, or the amount of execution-based selection used at test time.

This paper treats evaluation design as the object of study. The question is not whether structural priors are universally useful or useless. The question is which claim survives an audit that controls candidate budget, discloses information access, reports execution-selection assumptions, includes prompt-only boundary controls, and stratifies outcomes by diagnostic prior quality. We call the reporting pattern **SPC-Audit: Structural-Prior Claim Audit**. A central lesson is that structural-prior evaluation should report a quality response and claim-survival hierarchy, not only an average prior-versus-no-prior delta. The reusable output is an audit reporting checklist: budget accounting, execution-call accounting, prompt-token reporting, information-access disclosure, diagnostic-versus-deployable selection status, prompt-only controls, bad-prior controls, paired directionality, prior-quality diagnostics when reference structure is available, claim survival, and explicit non-claims.

The resulting picture is narrower than a method-success story, but more defensible as E&D. An uncontrolled `MBPP+224` comparison suggests a large gain, `167/224 -> 184/224`. Under fair-budget diagnostic comparison, the claim narrows to a modest effect in fixed code-aware `syntax_aware` episodes: `no_prior + MBR` solves `178.67 +/- 0.58`, while `multi_prior + MBR` solves `185.00 +/- 1.73`. Full `MBPP+224` prompt-only structural retrieval does not reproduce this positive effect. A prior-quality response audit is consistent with the boundary: intended priors are positive in medium/high diagnostic-fidelity bins, while the full prompt-only rerun rarely constructs high-fidelity priors. The contribution is therefore SPC-Audit and a claim-disciplined artifact, not a deployable structural-prior method.
